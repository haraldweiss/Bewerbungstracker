# Phase 4.2: Enhancement Features Design Specification

**Date:** 2026-04-22  
**Status:** Design  
**Scope:** Core editing + UX polish for iOS + Android  
**Priority:** High (blocking users from correcting data)  

---

## 1. Overview

Phase 4.2 adds three critical enhancements to Bewerbungstracker:

1. **Edit/Delete Applications** — Users can correct or remove application entries
2. **Dark Mode Support** — System-wide light/dark theme switching
3. **Email Detail View** — Read full email content without leaving the app

Phase 4.2 launches after Phase 5.1 iOS integration testing completes. Both iOS and Android implementations are identical in features and UX.

---

## 2. Feature 1: Edit/Delete Applications

### 2.1 UI/UX

**List View Changes:**
- Applications list cards now support **swipe or long-press** interactions
- Swipe left reveals: Edit | Delete buttons (iPhone style)
- Long-press shows action menu: Edit, Delete, Cancel
- Buttons: Edit (blue), Delete (red)

**Edit Screen:**
- Modal form (same as create application, but pre-filled)
- Title: "Edit Application"
- Fields:
  - Company (required, text input)
  - Position (required, text input)
  - Location (optional, text input)
  - Applied Date (date picker)
  - Notes (optional, text area)
- Buttons: Cancel, Save
- Save button disabled until at least one field changes
- Success: Toast notification "Application updated"

**Delete Confirmation:**
- Alert: "Delete Google — Software Engineer?"
- Subtext: "This cannot be undone."
- Buttons: Cancel (gray), Delete (red)
- Confirmation prevents accidental deletion

### 2.2 Data Flow

**Edit:**
```
User taps Edit card 
  → EditApplicationSheet opens with pre-filled data
  → User modifies company/position/location/date/notes
  → User taps Save
  → ViewModel calls updateApplication(id, request)
  → Database updates
  → List refreshes, card shows updated data
  → Toast: "Updated" appears
```

**Delete:**
```
User swipes left or long-presses
  → Delete button appears
  → User taps Delete
  → Confirmation alert shows
  → User taps Delete in alert
  → ViewModel calls deleteApplication(id)
  → Database removes record
  → List refreshes, card disappears
  → Toast: "Deleted" appears
```

**Code Example:**
```swift
// ApplicationsViewModel additions
@MainActor
func editApplication(id: UUID, company: String, position: String, 
                      location: String, appliedDate: Date, notes: String?) {
    let updatedApp = ApplicationModel(
        id: id,
        company: company,
        position: position,
        location: location,
        status: applications.first(where: { $0.id == id })?.status ?? .applied,
        appliedDate: appliedDate,
        createdAt: applications.first(where: { $0.id == id })?.createdAt ?? Date(),
        updatedAt: Date(),
        notes: notes
    )
    modelContext.insert(updatedApp)
    try? modelContext.save()
    fetchApplications()
}

@MainActor
func deleteApplication(id: UUID) {
    if let app = applications.first(where: { $0.id == id }) {
        modelContext.delete(app)
        try? modelContext.save()
        fetchApplications()
    }
}
```

## 3. Feature 2: Dark Mode Support

### 3.1 Design System

**Light Mode (Current):**
```
Background:          #FFFFFF
Card Background:     #FFFFFF
Text Primary:        #333333
Text Secondary:      #666666
Text Tertiary:       #999999
Border:              #E0E0E0

```

**Dark Mode (New):**
```
Background:          #1A1A1A
Card Background:     #2D2D2D
Text Primary:        #FFFFFF
Text Secondary:      #CCCCCC
Text Tertiary:       #999999
Border:              #404040
Section Background:  #242424
```

**Status Colors (unchanged in both modes):**
```

Applied:    #FF9800 (orange)
Offer:      #2196F3 (light blue)
Pending:    #9C27B0 (purple)
Primary:    #007AFF (blue)

```

### 3.2 iOS Implementation

**Files to Create:**
- `Utils/DarkModeColors.swift` — Dark mode color variants

**Files to Modify:**
- `Utils/Colors.swift` — Add @Environment detection for colorScheme
- All screen files — Update color references to use theme-aware colors

**Code Example:**
```swift
// Colors.swift
struct AppColors {
    @Environment(\.colorScheme) var colorScheme
    
    var background: Color {
        colorScheme == .dark ? Color(#colorLiteral(red: 0.1, green: 0.1, blue: 0.1, alpha: 1)) : .white
    }
    
    var cardBackground: Color {
        colorScheme == .dark ? Color(#colorLiteral(red: 0.18, green: 0.18, blue: 0.18, alpha: 1)) : .white
    }
    
    var textPrimary: Color {
        colorScheme == .dark ? .white : Color(#colorLiteral(red: 0.2, green: 0.2, blue: 0.2, alpha: 1))
    }
}
```

**Settings Toggle:**
- Add to Settings screen: "Appearance"
- Options: System, Light, Dark
- Store preference in UserDefaults
- Apply via `.preferredColorScheme()` modifier on AppDelegate

### 3.4 User Preference Storage

**iOS:** UserDefaults key `"darkModePreference"` (values: "system", "light", "dark")  
**Android:** DataStore key `"appearance_mode"` (values: "system", "light", "dark")

Default: "system" (follows device setting)

## 4. Feature 3: Email Detail View

### 4.1 UI Design

**Navigation:**
- From Emails tab: Tap email item → Slides to detail view
- Detail screen shows full email content
- Back button or swipe left to return to list

**Detail Screen Layout:**
```
[Back] [Subject: Interview Scheduled]

From:  recruiter@google.com
To:    user@example.com
Date:  April 20, 2026 • 2:22 PM (2 days ago)
━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Full email body with proper formatting]

Hello,

Your interview is scheduled for April 24 at 2:00 PM PST.

Please confirm your availability.

Best regards,
Google Recruiting
```

**Features:**
- Scrollable content (email body may be long)
- Preserve text formatting (paragraphs, line breaks)
- Strip HTML tags but keep structure (bold, italic, links)
- Show relative timestamp ("2 days ago")
- Back button in navigation

**Files to Create:**

struct EmailDetailView: View {

            VStack(alignment: .leading, spacing: 12) {

                            Text("Back")
                        }

                    Text("From: \(email.from)")
                        .font(.system(size: 12, weight: .regular))
                        .foregroundColor(.secondary)
                    Text(email.timestamp.formatted())
                        .font(.system(size: 12, weight: .regular))
                        .foregroundColor(.secondary)
                }
                
                Divider()
                
                Text(email.body.htmlToPlainText())
                    .font(.system(size: 14, weight: .regular))
                    .lineSpacing(6)
            }
            .padding(16)
        }
        .navigationBarBackButtonHidden(true)
    }
}
